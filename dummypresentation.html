OCR A Level Computer Science - Data Structures

Arrays

Definition: Arrays are a collection of elements of the same type, stored in contiguous memory locations.

  • Static Size: Once created, their size cannot be changed.
  • Mutable: Elements can be modified after creation.
  • Indexing: Elements are accessed via an index (starting from 0).

Use Cases: Efficient when you know the fixed number of elements.

Code Example:

# Python-like syntax
arr = [1, 2, 3, 4]
arr[0] = 10  # Mutable: we can change the value at index 0
print(arr)    # Output: [10, 2, 3, 4]

Tuples

Definition: An ordered collection of elements, can contain different types.

  • Static Size: Once created, their size cannot be changed.
  • Immutable: Cannot be modified after creation.
  • Accessing Elements: Indexed, just like arrays.

Use Cases: Ideal for constant data that shouldn't be modified.

Code Example:

# Python-like syntax
my_tuple = (1, "hello", 3.5)
# my_tuple[0] = 10  # Error: Tuples are immutable
print(my_tuple)  # Output: (1, "hello", 3.5)

Lists

Definition: Similar to arrays but more flexible, storing elements in an ordered collection.

  • Dynamic Size: The number of elements can change over time.
  • Mutable: Elements can be modified or removed.
  • Heterogeneous: Can store different types of data.

Use Cases: When the number of elements can vary and you need flexibility.

Code Example:

# Python-like syntax
my_list = [1, 2, 3, 4]
my_list.append(5)  # Adding an element dynamically
my_list[2] = 10  # Modifying an element (mutable)
print(my_list)  # Output: [1, 2, 10, 4, 5]

Multidimensional Arrays

Definition: Arrays of arrays, allowing for the creation of matrices or higher-dimensional structures.

  • Static Size: The size is fixed once created, though inner arrays may vary in some languages.
  • Mutable: Elements within the multidimensional array can be modified.
  • Indexed: Accessed via multiple indices (e.g., `array[row][column]`).

Use Cases: Useful for representing matrices or tabular data.

Code Example:

# Python-like syntax
matrix = [[1, 2, 3], [4, 5, 6], [7, 8, 9]]
matrix[1][2] = 10  # Modifying an element (mutable)
print(matrix)  # Output: [[1, 2, 3], [4, 5, 10], [7, 8, 9]]

Records (Structs/Dictionaries)

Definition: A collection of fields (attributes), each with a specific type and name, used to represent real-world entities.

  • Static Structure: The structure of the record is typically fixed.
  • Mutable: Fields within a record can be modified.
  • Heterogeneous: Records can store fields of different types.

Use Cases: Representing complex data with multiple attributes (e.g., a student or a product).

Code Example:

# Python-like syntax
student = {"name": "Alice", "age": 20, "grade": "A"}
student["age"] = 21  # Mutable: we can modify the age field
print(student)  # Output: {'name': 'Alice', 'age': 21, 'grade': 'A'}

Comparison Summary

Structure Static/Dynamic Mutable/Immutable Use Cases
Array Static Mutable Efficient for fixed size data
Tuple Static Immutable For constant data that shouldn't change
List Dynamic Mutable When the number of elements can change
Multidimensional Array Static Mutable For matrix-like or tabular data
Record (Struct/Dict) Static Mutable When dealing with structured data
